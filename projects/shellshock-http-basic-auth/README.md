# Shellshock and HTTP Basic Auth

## Scenario
This project is based on network-focused lab activities from Let's Defend related to Shellshock attack analysis and HTTP Basic Authentication investigation.

The exercises focused on suspicious HTTP traffic, packet analysis, server fingerprinting, and attacker behavior observed in PCAP files.

## Objective
The goal was to analyze packet captures, identify suspicious HTTP-based activity, understand how the attack worked, and extract useful information from the traffic.

## Tools Used
- Wireshark
- PCAP analysis
- HTTP traffic inspection
- Basic authentication analysis
- Base64 decoding concepts

## What I Did
- Analyzed HTTP traffic in packet captures
- Investigated suspicious requests related to a Shellshock-style attack
- Reviewed server and application details exposed through HTTP headers
- Examined HTTP Basic Authentication traffic and identified how credentials were transmitted
- Focused on understanding the attack flow and documenting technical findings

## Outcome
This project improved my confidence in PCAP analysis, HTTP investigation, and recognizing insecure authentication and web-based attack behavior.

## Skills Demonstrated
- PCAP analysis
- HTTP traffic analysis
- Shellshock investigation
- Basic authentication analysis
- Technical documentation
