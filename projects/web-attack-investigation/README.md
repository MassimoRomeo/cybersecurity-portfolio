# Web Attack Investigation

## Scenario
This project is based on the Let's Defend "Investigate Web Attack" challenge.

The investigation focused on suspicious activity found in web server access logs and on reconstructing the attack flow from the log evidence.

## Objective
The goal was to analyze web server logs, identify attacker behavior, recognize different stages of the attack, and document the findings clearly.

## Tools Used
- Web server access logs
- Browser-based investigation
- Log analysis methodology
- Basic web security knowledge

## What I Did
- Reviewed suspicious HTTP requests in the access log
- Identified reconnaissance activity and attacker enumeration behavior
- Analyzed signs of directory brute forcing and login abuse
- Tracked suspicious requests related to possible command or code injection
- Focused on reconstructing the attack chain step by step from the available evidence

## Outcome
This project helped me improve my ability to investigate web attacks through log analysis and to understand how multiple attack techniques can be chained together.

## Skills Demonstrated
- Web log analysis
- HTTP investigation
- Attack chain reconstruction
- Detection mindset
- Technical documentation
